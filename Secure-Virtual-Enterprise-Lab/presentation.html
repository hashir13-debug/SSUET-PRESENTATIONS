12 / Real Challenges & How We Solved Them

Problems Faced & Solved

⚡ Hypervisor Conflict — Hyper-V vs VMware (Biggest!)

Windows 11 Hyper-V grabs VT-x extensions — ESXi inside Workstation wouldn't boot. Gemini search visible in our screenshots!

bcdedit /set hypervisorlaunchtype off → disabled Hyper-V → restarted → ESXi booted

💾 No Datastore — Can't Create VMs

"No datastores configured" error on fresh ESXi install. ESXi needs VMFS storage partition before any VM can be created.

ESXi Host Client → Storage → New Datastore → VMFS6 → named "Main-Storage"

🌐 Windows Client Cannot Join Domain

"Domain could not be contacted" even with DC online. Root cause: DNS was pointing to pfSense, not the Domain Controller.

Changed client Primary DNS to 192.168.10.10 (DC-SRV01) → domain join succeeded instantly

🖥️ Only 4 GB RAM on One Machine

Couldn't run multiple VMs simultaneously — memory exceeded with Workstation + ESXi + VMs all running.

One VM at a time on the 4GB machine · heavier VMs on i7 · split work across both team laptops

⏱️ Windows Server Install — 45+ Minutes

Nested virtualisation + limited I/O = extremely slow setup. Visible at 47% in our actual screenshot.

Pre-staged ISOs on datastore · worked on other VMs in parallel to save time

🎩 RHEL 9 Subscription — Registration Required

Red Hat Enterprise Linux requires subscription registration before package updates. Initial network error "Name or service not known" blocked subscription-manager.

sudo subscription-manager register → registered system ID b380c0af → Registration Successful · RHEL updates enabled
