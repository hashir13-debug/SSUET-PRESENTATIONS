Setup in Class — Step by Step
01

Connect to Projector

Plug in HDMI or VGA cable to laptop. Press Win + P on keyboard. Select "Duplicate" — both your screen and the projector show the same thing.

If the projector shows nothing: unplug and re-plug the cable, then press Win + P again.
02

Open Presentation

Open Chrome → double-click presentation.html from Desktop (or drag-drop it into Chrome). Wait for all images to load — you'll see the animated particle background and Slide 1.

Press F11 for full screen. The entire slide should fill the screen — no browser address bar visible.

03

Test Navigation

Press → (right arrow) once — should go to Slide 2. Press ← (left arrow) — back to Slide 1. Works!

You can also click the "Next →" button at the bottom of each slide if keyboard is inconvenient.

Click on any screenshot image to zoom it full-screen. Press Escape to close the zoom. Show this to impress the class!
04

Start Presenting

Stand next to the screen. Face the audience (not the screen). Speak clearly. Use your laptop keyboard to navigate — you don't need to look at it.

Challenges ~2 min

Say this

Q&A — Common Professor Questions & Answers
Q
What is the difference between Type-1 and Type-2 hypervisor?
A
Type-1 hypervisor (like ESXi) installs directly on hardware — no OS underneath. It has direct access to CPU, RAM, storage. Type-2 hypervisor (like VMware Workstation, VirtualBox) runs ON TOP of an existing operating system like Windows. Type-1 is faster and used in production data centres. We used Type-2 (Workstation) to run Type-1 (ESXi) — this is nested virtualisation.
Q
What is a DMZ and why is it important?
A
DMZ stands for DeMilitarised Zone — it's a network segment that sits between the internet and the internal LAN. Servers that need to be publicly accessible (like web servers) are placed here. The key security property: DMZ cannot communicate back to LAN. So if an attacker compromises the web server, they are trapped in the DMZ — they cannot reach internal employee machines or the domain controller.
Q
What is pfSense and why did you choose it over a real firewall?
A
pfSense CE is an open-source firewall and router based on FreeBSD. It provides enterprise-grade features: VLAN routing, stateful firewall, NAT, DHCP, DNS, VPN support — completely free. We chose it because it's widely deployed in real SMB environments and provides the same features as paid firewalls like Cisco ASA, at zero cost. It's perfect for our virtual lab.
Q
What is Active Directory and how does it improve security?
A
Active Directory is Microsoft's centralised identity and access management system. Instead of each computer having its own local accounts, all users authenticate against a central Domain Controller. This means: one account works on every domain computer, password policies are enforced everywhere centrally, Group Policy Objects can restrict what users can do (install software, change settings), and all login events are logged centrally for auditing.
Q
What is nested virtualisation? Is it slower?
A
Nested virtualisation means running a hypervisor (ESXi) as a virtual machine inside another hypervisor (Workstation). Yes, it is slower than running ESXi on real hardware — we experienced this when Windows Server took 45+ minutes to install instead of the usual 10 minutes. This is because each CPU instruction goes through two layers of virtualisation. However, for learning and testing purposes, it works perfectly. In production, ESXi would run on a real server.
Q
How did you test the firewall rules?
A
We used Kali Linux placed in the Management zone as our testing machine. We ran Nmap network scans and ping tests from different zones. For example, to test that DMZ cannot reach LAN, we ran "ping 192.168.10.10" from the Ubuntu web server in DMZ — the ping timed out, confirming the rule works. We also ran "nmap -sT 192.168.10.10" to test all ports — all were filtered, confirming the block.
Q
What is Fail2Ban?
A
Fail2Ban is an intrusion prevention software for Linux. It monitors log files (like /var/log/auth.log for SSH) and automatically bans IP addresses that show signs of brute-force — for example, 5 failed SSH login attempts within a short time. The IP is added to the firewall's drop list using iptables or UFW. This protects our web server from automated password-guessing attacks.
Q
What would you improve if you had more time?
A
We identified several improvements: First, adding a SIEM (Security Information and Event Management) system like Wazuh to centralise logs from all machines in one dashboard. Second, implementing HTTPS with SSL/TLS certificates on the web server. Third, adding an IDS/IPS like Snort on pfSense for real-time intrusion detection. Fourth, deploying OpenVPN for secure remote administration. These would make the lab even closer to a production enterprise environment.

Pro Tips — Stand Out in Class
🎤

Speak Slowly & Clearly

Professors judge confidence. Speak 20% slower than you think you need to. Pauses are good — they show you're thinking, not memorising.

👀

Face the Audience

Don't face the screen. Turn your body to the audience. Use your hand to point backward at the screen while looking forward at the class.

📌

Point at Specific Things

When showing screenshots: point at EXACT details. "Dekho is screenshot mai 23ms likha hai — ye internet connection confirm karta hai." Specificity = credibility.

❓

Q&A Strategy

If you don't know the answer: "That's a great question. I'd like to look into that more — but from what I know..." Never just say "I don't know" without attempting an answer.

💬

Use Technical Terms

Say: "deny-by-default policy", "stateful firewall", "nested virtualisation", "zone isolation". These terms show you understand the concepts, not just the steps.

🏆

Highlight Real Proof

Keep saying: "Ye humne actually implement kiya." Point to screenshots. "Ye real output hai — fake nahi." Real evidence is your strongest asset.

🔍

Use the Zoom Feature

When showing screenshots: CLICK on them to zoom full-screen. This shows the class real terminal output, real IP addresses, real system details. Very impressive.

🤝

Team Handoffs

When switching presenter: "Ab [name] explain karey ga..." (Now [name] will explain...). Clean handoffs show professionalism and teamwork.

Emergency — If Something Goes Wrong

🚫 Presentation file won't open

The file only needs Chrome or Edge to open. No internet required — everything is local. Right-click the file → Open With → Google Chrome.

Keep a copy on a USB drive as backup. If everything fails, open report.html — it has all the content in document form.

🖥️ Projector not showing anything

Press Win + P → Duplicate. If that doesn't work, unplug and re-plug the HDMI cable. Try the projector's Input/Source button to select the correct HDMI port.

If projector completely fails, present from your laptop screen while holding it up, or bring the class around the laptop. Not ideal but works.

📷 Screenshots not loading (broken image)

This happens if the .jpeg image files are not in the same folder as presentation.html. They must be in the same directory.

Copy all 9 WhatsApp .jpeg files into the same folder as presentation.html. Refresh the page (F5). They will appear.

⌨️ Arrow keys not working

Click anywhere on the slide once to give the page focus, then try arrow keys. Or use the Next → button at the bottom of each slide.

Touch swipe also works on touch screens — swipe left to go forward, right to go back.

😰 You forget what to say mid-presentation

Pause. Take a breath. Look at the slide. Every slide has its title as a hint for what to talk about. Read one bullet point and explain it — buy yourself time.

Say "Let me recap this point..." — this gives you time to remember. Or hand off to your teammate: "Hashir, do you want to add anything?" Teamwork saves the day.

🤔 Professor asks something you weren't prepared for

It's completely fine to say: "We didn't specifically test that scenario, but based on our implementation, I believe [your best answer]..."

Then follow up: "I'll look into that and can provide a more detailed answer after class." Professors respect intellectual honesty.
