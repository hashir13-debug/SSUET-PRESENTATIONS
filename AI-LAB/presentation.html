01 · The Idea

What is MedCore AI?

  • MedCore AI is a fully working, localized, AI-powered health diagnostic web app — built as a hands-on exploration of applied AI.
  • It's an accessible first step for anyone feeling unwell — no appointment, no waiting room, available instantly.
  • The user describes how they feel and where they live; the system replies with a preliminary diagnosis, the right specialist type nearby, and the estimated medicine cost in their own currency.
⚠  Important: MedCore AI positions itself strictly as a triage and educational tool — guiding decisions, never replacing a real doctor.

03 · System Architecture

From Symptom to Diagnosis — In 5 Steps

1User Input — symptoms and location are typed into the React frontend.
2Secure Transmission — data travels safely to the Next.js backend; no API keys ever touch the browser.
3Prompt Engineering — the backend wraps the data in a strict "System Prompt," instructing the 72B model to act as a medical expert and reply in rigid JSON.
4AI Inference — the Hugging Face router returns the diagnosis, local doctor recommendations, and medicine price estimates.
5Dynamic Rendering — the frontend parses the JSON and builds a polished, tabbed results UI on the fly.
04 · Real Engineering Challenges

Obstacle #1 & #2 — and How We Beat Them

Problem

🚫 Hard ISP DNS Blocks

The official Hugging Face API domain was completely unreachable — every call failed with ERR_NAME_NOT_RESOLVED, as if blocked at the DNS level.

Our Fix

Switched to the official @huggingface/inference SDK, which auto-reroutes traffic through router.huggingface.co to alternate providers — bypassing the blackout entirely.

Problem

⚠️ Model Rejected (HTTP 400)

Smaller models like Mistral 7B kept failing with 400 Bad Request — the new routing infrastructure no longer supported chat-completion for them on the free tier.

Our Fix

Upgraded to Qwen2.5-72B-Instruct — natively supported, which fixed the crash and made the diagnoses noticeably sharper.

04 · Real Engineering Challenges

Obstacle #3 & #4 — and How We Beat Them

Problem

💥 React Crashed on AI Output

The 72B model sometimes returned deeply nested JSON (objects inside arrays) instead of plain text — triggering React's fatal "Objects are not valid as a React child" error.

Our Fix

Wrote a recursive renderContent function that detects nested data and turns it into clean, styled lists — converting a guaranteed crash into a polished "Diagnostic Card" feature.

Problem

⌨️ Terminal Was Corrupting Code

Pasting CSS through Windows PowerShell silently replaced real line breaks with literal \n text, breaking the stylesheet with hidden syntax errors.

Our Fix

Bypassed the terminal entirely — edited the CSS files directly through safe, structured file-editing tools, restoring clean formatting instantly.

05 · Why It Matters

Key Highlights

  • Zero cost — the entire AI pipeline runs on Hugging Face's free-tier inference; nothing was paid for.
  • Premium design — a Glassmorphism UI (frosted-glass panels, soft gradients, micro-animations) gives it a polished, professional feel.
  • Real localization — the AI reasons about the user's actual city to suggest realistic hospitals and convert medicine prices into local currency.
  • Resilient by design — even when the AI returns messy, nested data, the app gracefully turns it into a clean, readable interface instead of crashing.
🚀  A small student project that quietly solves real infrastructure problems — DNS blackouts, model limits, rendering crashes — the same way production teams do.
