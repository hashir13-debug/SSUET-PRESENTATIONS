01Project Overview

MedCore AI is a fully functional, localized, AI-powered health diagnostic web application. It serves as an accessible first step for users experiencing medical symptoms: by simply describing how they feel and where they live, the system returns a preliminary diagnosis, recommends the precise type of specialist to consult in their city, and estimates the cost of over-the-counter relief medicines in their local currency.

⚠ Disclaimer — The platform is explicitly framed as a triage and educational tool, and not a replacement for professional medical advice.

02Target Audience

  • General Public — individuals seeking a quick, preliminary understanding of their symptoms.
  • Patients in Remote Areas — people who need immediate guidance on whether to see a doctor, and exactly which kind of doctor to visit locally.
  • Students & Academics — a working demonstration of practical, applied AI in the healthcare domain.

04System Architecture — How It Works

  1. User Input — the user types their symptoms and location into the React frontend.
  2. Secure Transmission — the data is sent securely to the Next.js backend; API keys are never exposed in the browser.
  3. Prompt Engineering — the backend wraps the user's data inside a strict "System Prompt," instructing the 72B model to act as a medical expert, localize its reasoning to the user's city, and respond strictly in a rigid JSON format.
  4. AI Inference — the Hugging Face inference router processes the request and returns the diagnosis, recommended local doctors, and estimated medicine prices.
  5. Dynamic Rendering — the frontend parses this JSON object and dynamically generates a tabbed results interface without crashing.

05Major Problems Faced & Technical Solutions

During development, four significant technical hurdles emerged. Each was diagnosed methodically and engineered around successfully:

PROBLEM 1

Hard ISP DNS Blocks

The Issue — the standard Hugging Face API domain (api-inference.huggingface.co) was globally unreachable, producing ERR_NAME_NOT_RESOLVED network failures, as though local ISPs or DNS root zones had blocked or deprecated the domain.

SOLUTION

We migrated the architecture to the official @huggingface/inference Node SDK on the server side. This SDK contains internal routing logic that automatically bypasses blocked domains by directing traffic through router.huggingface.co to third-party serverless providers (such as Novita) — completely resolving the connectivity blackout.

PROBLEM 2

Model Capability Restrictions (HTTP 400 Errors)

The Issue — due to the new routing infrastructure, smaller models (such as Mistral 7B) returned 400 Bad Request errors, since free-tier providers no longer supported the chatCompletions task for them.

SOLUTION

We upgraded the application to Qwen2.5-72B-Instruct, a 72-billion-parameter model natively supported by the new free-tier infrastructure — eliminating the crash while substantially improving medical reasoning accuracy.

PROBLEM 3

Complex JSON React Rendering Crashes

The Issue — because the 72B model is highly capable, it occasionally returned deeply nested JSON objects and arrays (e.g. [{condition: "Flu", explanation: "..."}]) instead of simple flat strings. When React attempted to render these objects directly as text, it triggered the fatal "Objects are not valid as a React child" crash.

SOLUTION

We engineered a recursive renderContent mapping function on the frontend. When the AI returns complex nested objects, this function detects them and maps them into styled <ul> / <li> HTML lists — converting a potential crash into a polished "Diagnostic Card" UI feature.

PROBLEM 4

Terminal Encoding Corrupting Code

The Issue — while injecting CSS for the Tabbed UI through the Windows PowerShell terminal, its text encoding inserted literal \n characters in place of real line breaks, causing a fatal CSS syntax error that broke the styling engine.

SOLUTION

The file system was accessed directly through secure code-replacement tools to rewrite the CSS blocks correctly — completely bypassing the terminal's encoding issue.

06Exhibition Talking Points

  • Emphasize that zero dollars were spent building or running this — AI inference runs entirely on Hugging Face's open-source free tier.
  • Highlight the Glassmorphism UI — it presents like a premium product rather than a typical lab assignment.
  • Point out the Localization Feature — the AI calculates medicine prices in PKR (or the relevant local currency) and recommends real hospital names in the user's city, requiring genuine LLM reasoning.
  • Mention the Resilient Architecture — explain how the application safely handles complex nested JSON from the AI and maps it dynamically to the UI without crashing.
